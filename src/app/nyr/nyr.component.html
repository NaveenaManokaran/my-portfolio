The challenge: Cisco Systems, Inc. have a lot of different applications in which a task may wait for another task to get completed, so, that output will be taken as input by it. Without holding if it executes it provides wrong outcome. The proposed concept of handling this feature is a New Year relevance tool where the child task should wait until the parent task is completed, in order to avoid inaccurate data input for execution.

The solution: Front end developed using HTML5, CSS3, Bootstrap and jQuery to get a responsive layout with the customized design for the website. Since they have an enormous amount of Task for each application, tables has been used for differentiating it easily. Popup have been Displayed for showing the details about the current taskâ€™s parent. This helps us to manage time Efficiently.
New desktop website containing all application details to avoid messy execution of the task when it is Having a dependency with more features.

The result: The new Year relevance tool contains the list of applications along With their task details, task dependent and task parameter in order to execute it, and also execution Status. Features include create, edit, view, search, export, and copy from etc.
Create and edit option available for application, a task where a view is applicable to see parent task, Completed task, scheduled task, status report, and also execution history. The popup is provided to show The linked parent task for current task.
